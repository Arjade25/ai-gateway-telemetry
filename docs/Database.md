# Database Design

Version: MVP

---

# Overview

The AI Gateway Telemetry Service persists one record for every successful AI interaction.

The MVP intentionally uses a simple relational data model.

There is **one primary table**.

No relationships.

No users.

No organizations.

No teams.

No multi-tenancy.

The objective is to reliably capture telemetry.

---

# Database Technology

Database

* PostgreSQL

ORM

* Prisma

The application should remain database agnostic where practical, but PostgreSQL is the official supported database for the MVP.

---

# Primary Entity

## ChatEvent

Represents a single request made to an LLM provider.

One request

↓

One telemetry record

There should never be multiple telemetry records for the same completed request.

# Schema

## id

Type

UUID

Description

Primary key.

Generated by the application.

Immutable.

---

## provider

Type

String

Examples

* anthropic
* openai
* google
* ollama

Purpose

Identifies which provider processed the request.

---

## model

Type

String

Examples

* claude-sonnet-4
* gpt-5
* gemini-2.5-pro

Purpose

Stores the exact model used.

---

## prompt

Type

Text

Purpose

The original prompt submitted by the client.

The gateway stores the raw prompt.

Future versions may support encryption or redaction.

---

## response

Type

Text

Purpose

The response returned by the provider.

Future versions may allow this to be disabled.

---

## input_tokens

Type

Integer

Purpose

Prompt token count.

---

## output_tokens

Type

Integer

Purpose

Response token count.

---

## total_tokens

Type

Integer

Purpose

Computed value.

```
total_tokens =

input_tokens

+

output_tokens
```

---

## latency_ms

Type

Integer

Purpose

Round-trip request latency.

Measured in milliseconds.

---

## cost

Type

Decimal

Purpose

Computed request cost.

Precision should support small fractional values.

Example

```
0.000217
```

Cost is derived from pricing configuration.

Never entered manually.

---

## finish_reason

Type

String

Examples

* stop
* max_tokens
* tool_use
* length

Provider dependent.

May be null if unavailable.

---

## provider_request_id

Type

String

Purpose

Unique identifier returned by the provider.

Useful for debugging.

May be null.

---

## created_at

Type

Timestamp

Purpose

Time the request completed.

Stored in UTC.

Generated by the server.

---

# Data Integrity

The application should guarantee

```
total_tokens =

input_tokens

+

output_tokens
```

This value should never be manually supplied by clients.

---

Cost should always be calculated by the gateway.

Clients must never send cost values.

---

created_at should always be generated by the server.

---

# Data Ownership

Only the AI Gateway Telemetry Service writes to this table.

Clients cannot modify telemetry records.

Future versions may support updates for data correction, but MVP records are immutable.

* organizations
* teams
* sessions
* conversations
* prompt categories

* metadata blobs

# Design Philosophy

The database should remain intentionally small.

The objective is not to model every aspect of AI interactions.

The objective is to create a reliable, consistent, and provider-agnostic telemetry record for every completed request.

Any new field should answer one question:

> Does this improve our ability to understand AI usage telemetry?

If not, it does not belong in the MVP.
